122 ## 3.0.0 beta 1 (2011-08-14)
123 
124 Breaking Changes:
125 * Removed overloads of logging methods that were taking format string from ILogger and ILogger and IExtendedLogger and didn't have word Format in their name.
126  * For example:
127  * void Error(string format, params object[] args); // was removed
128  * void ErrorFormat(string format, params object[] args); //use this one instead
129  * impact - low
130  * fixability - medium
131  * description - To minimize confusion and duplication those methods were removed.
132  * fix - Use methods that have explicit "Format" word in their name and same signature.
133 * Removed WebLogger and WebLoggerFactory
134  * impact - low
135  * fixability - medium
136  * description - To minimize management overhead the classes were removed so that only single Client Profile version of Castle.Core can be distributed.
137  * fix - You can use NLog or Log4Net web logger integration, or reuse implementation of existing web logger and use it as a custom logger.
138 * Removed obsolete overload of ProxyGenerator.CreateClassProxy
139  * impact - low
140  * fixability - trivial
141  * description - Deprecated overload of ProxyGenerator.CreateClassProxy was removed to keep the method consistent with other methods and to remove confusion
142  * fix - whenever removed overload was used, use one of the other overloads.
143 * IProxyGenerationHook.NonVirtualMemberNotification method was renamed
144  * impact - high
145  * fixability - easy
146  * description - to accommodate class proxies with target method NonVirtualMemberNotification on IProxyGenerationHook type was renamed to more accurate
147  NonProxyableMemberNotification since for class proxies with target not just methods but also fields and other member that break the abstraction will
148  be passed to this method.
149  * fix - whenever NonVirtualMemberNotification is used/implemented change the method name to
150  NonProxyableMemberNotification. Implementors should also accommodate possibility that not
151  only MethodInfos will be passed as method's second parameter.
152 * DynamicProxy will now allow to intercept members of System.Object
153  * impact - very low
154  * fixability - easy
155  * description - to allow scenarios like mocking of System.Object members, DynamicProxy will not
156  disallow proxying of these methods anymore. AllMethodsHook (default IProxyGenerationHook)
157  will still filter them out though.
158  * fix - whenever custom IProxyGenerationHook is used, user should account for System.Object's
159  members being now passed to ShouldInterceptMethod and NonVirtualMemberNotification methods
160  and if neccessary update the code to handle them appropriately.
